# visionone_cam_gcp_project_migration (Resource)

Migrates an existing GCP project record in the CAM database from the legacy Terraform Package Solution to the new Terraform Provider Solution. This updates the service account key information in the DB record so that downstream apps remain connected. Migration is a one-way operation: Update and Delete are no-ops.

## Use Cases

- **Legacy to Provider Migration**: Update the CAM database record to use the new service account key without downtime
- **Zero-Downtime Migration**: Runs while legacy resources are still active, ensuring no connectivity gap
- **In-Place Record Update**: Preserves the existing cloud account record so downstream services (alerts, policies) remain intact

## Behavior

- **`terraform apply`**: Updates the CAM database record to point to the new service account
- **`terraform destroy`**: No-op — the migration record is removed from state only, the CAM record is unchanged
- **`terraform update`**: No-op — migration is a one-way operation

## Example Usage

```terraform
# Migrate a GCP project from legacy Terraform Package Solution to Terraform Provider Solution.
# Step 2 of 3: update the CAM database record to use the new service account key.
resource "visionone_cam_gcp_project_migration" "example" {
  project_number          = "123456789012"
  name                    = "My GCP Connector"
  new_service_account_id  = visionone_cam_service_account_integration.new_sa.service_account_unique_id
  new_service_account_key = visionone_cam_service_account_integration.new_sa.private_key
}
```

### Required

- `name` (String) Display name for the connector (preserved during migration).
- `new_service_account_id` (String) The unique ID of the new service account created by the Terraform Provider Solution.
- `new_service_account_key` (String, Sensitive) Base64-encoded JSON service account key for the new Terraform Provider Solution service account.
- `project_number` (String) The GCP project number identifying the existing CAM record to migrate.

### Read-Only

- `id` (String) The ID of this resource.
- `migrated_at` (String) RFC3339 timestamp when the migration was performed.
- `migration_error` (String) Error message if the migration failed.
- `migration_status` (String) Status of the migration: `migrated` or `failed`.

## Important Notes

- **One-Way Operation**: Migration cannot be reversed via Terraform
- **Prerequisite**: The new service account and connector must exist before running migration (use `depends_on`)
- **No Delete**: Destroy only removes the Terraform state entry; it does not roll back the CAM database

## Import

Migration resources do not support import as they represent a point-in-time operation.
