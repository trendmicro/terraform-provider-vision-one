# visionone_cam_legacy_cleanup_service_account (Resource)

Deletes the legacy GCP service account (`vision-one-service-account`) and all its keys, as created by the Terraform Package Solution. Returns `cleanup_status = "not_found"` if the service account does not exist.

## Use Cases

- **Legacy to Provider Migration**: Remove the legacy service account after the CAM database has been updated to the new SA
- **Key Cleanup**: Automatically deletes all user-managed keys attached to the service account
- **Safe Cleanup**: Returns `not_found` if the service account does not exist (no errors)

## Behavior

- **`terraform apply`**: Detects `vision-one-service-account@{project_id}.iam.gserviceaccount.com`, deletes all its keys, then deletes the account
- **`terraform destroy`**: Removes resource from Terraform state only (service account unchanged)

## Example Usage

```terraform
# Delete the legacy service account (vision-one-service-account) and all its keys.
resource "visionone_cam_legacy_cleanup_service_account" "example" {
  project_id          = "my-gcp-project-id"
  service_account_key = var.legacy_service_account_key
}
```

## Required Permissions

The authenticating principal must have the following GCP permissions on the target project:

- `iam.serviceAccounts.get`
- `iam.serviceAccountKeys.list`
- `iam.serviceAccountKeys.delete`
- `iam.serviceAccounts.delete`

## Cleanup Status Values

- **`deleted`** — Legacy service account was found and deleted
- **`not_found`** — Service account does not exist (nothing to clean up)
- **`failed`** — Cleanup encountered an error (check `cleanup_error` field)

## Import

Legacy cleanup resources do not support import as they are cleanup operations rather than persistent GCP resources.
