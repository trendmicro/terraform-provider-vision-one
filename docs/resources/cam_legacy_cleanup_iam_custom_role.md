# visionone_cam_legacy_cleanup_iam_custom_role (Resource)

Deletes the legacy IAM custom role (`vision_one_cam_role_*`) created by the Terraform Package Solution. The resource scans all project-level custom roles for the naming prefix and deletes the match. Returns `cleanup_status = "not_found"` if no legacy role exists.

## Use Cases

- **Legacy to Provider Migration**: Remove the legacy IAM custom role after deploying the new provider-managed role
- **Prefix-Based Detection**: Automatically scans for `vision_one_cam_role_*` without requiring the exact role ID
- **Safe Cleanup**: Returns `not_found` if no legacy role exists (no errors)

## Behavior

- **`terraform apply`**: Scans project-level custom roles for the `vision_one_cam_role_` prefix and deletes the match
- **`terraform destroy`**: Removes resource from Terraform state only (IAM role unchanged)
- **Skip New Role** (`custom_role_id`): If set, any role with that exact name is skipped during prefix scan — prevents accidental deletion of the new provider-managed role

## Example Usage

```terraform
# Delete legacy IAM custom role.
resource "visionone_cam_legacy_cleanup_iam_custom_role" "example" {
  project_id          = "my-gcp-project-id"
  service_account_key = var.legacy_service_account_key

  # Optional: skip new provider-managed role.
  # custom_role_id = visionone_cam_iam_custom_role.new_role.name
}
```
